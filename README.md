# 📚 अवधी AI Physics Vernacular Tutor

An interactive, vernacular voice-to-voice AI physics tutor for UP Board students, featuring a Coordinator Dashboard to design custom lesson roadmaps, real-time student profiling, interactive quizzes, and automatic diagnostic markdown reports.

---

## 👥 Team & Project Details
* **Team Owner**: `@ethicalvats` (GitHub)
* **Project Title**: **अवधी AI Physics Vernacular Tutor**
* **Short Description**: Awadhi AI Physics Tutor is a low-latency, voice-to-voice educational assistant designed for UP Board students, speaking in the local Lucknowi Awadhi dialect. It features a dual-portal design: a **Coordinator Portal** for teachers to customize tutor settings, set lesson checklists, and monitor student profiling logs, and a **Student Portal** for classroom learning with interactive, sandboxed physics blackboard animations, interactive quizzes, and voice/chat capabilities.

---

## 💡 Problem Statement (PS-03)
Lakhs of UP Board students prepare in Hindi/Awadhi with little access to quality coaching (e.g., the Hazratganj coaching belt). This project builds an agentic tutor that:
1. Identifies student learning gaps and strengths dynamically.
2. Generates practice questions from past board papers as interactive quizzes.
3. Adapts to each student's chosen dialect (Awadhi, Hindi, or Auto-Adapt) and pace (Slow, Normal, or Fast).
4. Reports student diagnostic progress in real-time to a coordinator and archives logs to disk.

---

## 🏗️ Portal Architecture & Features

### 1. 🏫 Coordinator Hub (Hazratganj Admin Portal) - `/coordinator`
* **API Key Synchronization**: Securely registers and stores Gemini API keys on the server filesystem to auto-authenticate students.
* **Tutor Template Designer**: 
  - Define custom tutor templates, custom Dialect, and Speed constraints.
  - **Lesson Plan Builder**: Add, edit, or remove step-by-step lesson checklist items.
* **Student Registry**: Register new student profiles, delete profiles, and assign specific tutor templates to students (1-to-1 mapping).
* **Live Student Monitor (2.5s Polling loop)**:
  - Live progress tracking along the lesson checklist.
  - **Diagnosed Gaps & Weaknesses**: Red pill tags identifying topics the student struggled with.
  - **Mastered Strengths**: Green pill tags representing concepts the student answered correctly.
  - **Quiz Performance Score**: Circular progress tracker of quiz percentages.
  - **Diagnostics Summary**: Real-time qualitative assessment text generated by the AI tutor.
  - **Export Markdown Report**: Save structured diagnostic profiles to `/reports/` on the local file system.
  - **Reset Progress & Lesson**: Revert student lesson progress, clear gaps/strengths, and reset quiz score to rerun lessons.

### 2. 🎓 Student Classroom - `/student`
* **Classroom Selector**: Card gallery to choose registered students.
* **Checklist Roadmap**: Displays the assigned tutor's lesson plan. Highlights completed, active, and upcoming steps.
* **Voice & Text Classroom**: Low-latency multimodal voice connection with microphone input, audio playback, transcript viewer, and barge-in support.
* **Speech-Wave Visualizer**: Fluid sound-wave animation bars inside the avatar container to display active mic/audio connections.
* **Interactive Blackboard & Quiz Card**:
  - Automatically renders custom, interactive HTML5/JS physics canvas visuals generated on-demand by the tutor.
  - Displays beautiful, glassmorphic interactive quiz cards with options when the tutor asks a physics MCQ.
  - Renders correct (`✅`) and incorrect (`❌`) feedback styling on button click, adjusts score, and submits answer via WebSocket `toolResponse` to Gemini.
* **Quick Actions**: Buttons to prompt the tutor to proceed to the next lesson step or start a past-paper quiz instantly.
* **Reset Progress**: Allows the student to reset their lessons and start fresh from the classroom setup.

## 🛠️ Tech Stack & Tools
* **Frontend**: React 19, Vite, Vanilla CSS (Premium Glassmorphism Dark-Space Theme)
* **Audio Pipeline**: Web Audio API (PCM 16kHz Microphone Recorder, PCM 24kHz Audio Player with barge-in interruption)
* **WebSocket Channel**: Gemini Multimodal Live API WebSocket (`wss://`)
* **AI Model Engine**: `models/gemini-3.1-flash-live-preview` (supporting bidirectional live streaming and tool calling)
* **Server Middlewares (Node/Vite)**: 
  - `GET /api/state`: Fetches registered tutors, students, and API key.
  - `POST /api/update-student-diagnostics`: Updates student progress metrics (score, gaps, strengths).
  - `POST /api/save-logs`: Saves WebSocket raw message transaction history under `/logs/`.
  - `POST /api/save-report`: Saves student diagnostic profiles as markdown files under `/reports/`.

## 🚀 Setup & Execution Instructions

### Prerequisites
* **Node.js**: Version 18 or above
* **npm**: Version 9 or above
* **Gemini API Key**: Obtain a key from Google AI Studio.

### Steps to Run
1. **Clone the Repository**:
   ```bash
   git clone https://github.com/ethicalvats/<repo-name>.git
   cd lucknow-gdg-hack
   ```

2. **Install Dependencies**:
   ```bash
   npm install
   ```

3. **Start Development Server**:
   ```bash
   npm run dev
   ```

4. **Access the App**:
   - Open **Browser A** at `http://localhost:5173/coordinator` to manage tutors and students.
   - Open **Browser B** at `http://localhost:5173/student` (or another tab/window) to experience the student portal.
   *Note: Because routing uses standard URL pathnames and state is persisted on the server, you can run the student and coordinator panels in completely different browsers (e.g. Chrome and Safari) without session pollution!*

5. **Usage Guide**:
   - Paste your **Gemini API Key** in the Coordinator Hub settings input.
   - Create a custom Tutor Template (e.g. "Gomti Gravity Guide" in Awadhi).
   - Register a student and assign the created tutor.
   - Switch to the **Student Portal**, select the student, click **Reset Progress** if needed, and click **Connect**.
   - Speak or click **Take Quiz** / **Next Step** to interact. Watch the coordinator view update live every 2.5s!

## ⚠️ Known Limitations & Incomplete Features
* **Browser Sandbox Restrictions**: The visual blackboard is isolated using `sandbox="allow-scripts"`. Any script crash inside the AI-generated visual executes securely in its own scope and does not crash the host app.
* **Dialect Drift**: Since Gemini models are trained broadly, the model may occasionally slip from Lucknowi Awadhi into standard Hindi when expressing extremely complex English terminology (e.g., "Coefficient of Friction").
* **Local Reports Storage**: The automatic local file-writing server-side endpoint works exclusively when running the local Vite server (`npm run dev`).
